# Задания для Агента 2: Frontend Бизнес-логика

Проверь общий файл common.txt для общей информации о проекте.

## Твоя роль
Ты отвечаешь за бизнес-логику на стороне клиента и управление состоянием приложения в онлайн-магазине цветов "Флора".

## Твои зоны ответственности
1. Настройка и управление Redux состоянием
2. Взаимодействие с API бэкенда
3. Обработка данных и бизнес-логика на клиенте
4. Валидация форм
5. Аутентификация и авторизация на фронтенде
6. Обработка ошибок и мониторинг
7. Оптимизация производительности приложения

## Первые задачи
1. Настройка Redux:
   - Настроить хранилище с redux-toolkit
   - Имплементировать слайсы для товаров, корзины, пользователя
   - Создать асинхронные thunks для взаимодействия с API

2. Реализовать логику для:
   - Загрузки и фильтрации каталога товаров
   - Добавления/удаления товаров в корзине
   - Процесса оформления заказа
   - Авторизации и регистрации пользователей

3. Валидация и формы:
   - Реализовать валидацию для форм заказа и авторизации
   - Создать обработчики ошибок и обратную связь пользователю

## Технические требования
- Использовать Redux Toolkit для организации состояния
- Применять RTK Query для взаимодействия с API
- Писать типизированный код (TypeScript)
- Следовать принципам чистого кода и переиспользования
- Покрывать логику тестами

## Взаимодействие с другими разработчиками
- Агент 1 создаст UI компоненты, которые ты будешь соединять с Redux
- Агент 3 разрабатывает API бэкенда, с которым тебе нужно взаимодействовать
- Координируй API контракты с Агентом 3
