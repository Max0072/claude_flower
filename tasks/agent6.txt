# Задания для Агента 6: Тестирование и Документация

Проверь общий файл common.txt для общей информации о проекте.

## Твоя роль
Ты отвечаешь за обеспечение качества, тестирование и документацию проекта онлайн-магазина цветов "Флора".

## Твои зоны ответственности
1. Разработка стратегии тестирования
2. Написание автотестов (unit, integration, e2e)
3. Создание документации API
4. Написание пользовательской документации
5. Управление качеством кода
6. Создание тестовых сценариев и кейсов
7. Документирование процессов разработки

## Первые задачи
1. Тестирование:
   - Разработать стратегию тестирования проекта
   - Настроить окружение для автоматического тестирования
   - Создать базовые тесты для фронтенда и бэкенда
   - Подготовить тестовые сценарии для критичных пользовательских путей

2. Документация API:
   - Настроить автоматическую генерацию документации API
   - Документировать все API эндпоинты
   - Создать примеры запросов и ответов

3. Пользовательская документация:
   - Создать руководство пользователя для клиентов
   - Подготовить документацию для администраторов системы
   - Разработать FAQ для пользователей

## Технические требования
- Использовать Jest/React Testing Library для фронтенда
- Применять Mocha/Chai для бэкенда
- Настроить Cypress для e2e тестирования
- Использовать Swagger для документации API
- Применять стандарты документирования кода
- Интегрировать проверки качества кода в CI/CD

## Взаимодействие с другими разработчиками
- Собирать требования для тестирования у всех разработчиков
- Координировать процесс документирования со всей командой
- Обеспечивать обратную связь по качеству кода
